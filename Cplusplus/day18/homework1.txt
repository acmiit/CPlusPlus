STL包含那些组件？各自具有那些特点
一、STL包含六大组件
分别为：容器、迭代器、算法、适配器、函数对象、分配器

1、容器：用来存储和管理数据集合的类模板
容器分为：序列式容器(vector、deque、list)、关联式容器(map、set)、无序关联式容器(unordered_set、unordered_map)
（1）序列式容器：元素顺序与插入顺序一致
vector：动态数组，支持快速随机访问，尾部插入删除高效，中间插入删除较慢
deque：双端队列，支持头尾快速插入删除，随机访问较快
list：双向链表，任意位置插入删除高效，不支持随机访问
forward_list：单向链表，更节省空间，但只能单向遍历
array：固定大小数组，支持快速随机访问，不能改变大小
（2）关联式容器：元素按关键字有序保存
set：基于红黑树实现，关键字即值，不允许重复，自动排序
map：基于红黑树实现，键值对，键唯一，自动按键排序
multiset：关键字可重复的set
multimap：键可重复的map
（3）无序关联式容器
unordered_set：基于哈希表，平均o(1)查找、插入、删除，最坏o(n)性能
unordered_map：键无序，平均o(1)操作，需要自定义类型的哈希函数
unordered_multiset：支持重复元素，其他同无序版本
unordered_multimap：支持重复元素，其他同无序版本

2、迭代器：连接容器和算法的桥梁
迭代器提供了一种统一的方法来遍历容器中的元素，类似于指针。分为：
输入迭代器：只读，支持++、==、!=、*。
输出迭代器：只写，支持++、*。
前向迭代器：可读写，支持++、==、!=、*，可多次遍历。
双向迭代器：在前向迭代器基础上增加--操作。
随机访问迭代器：支持迭代器算术运算（+、-、+=、-=、[]），类似于指针

3、算法:处理容器的函数
头文件：#include<algorithm>
STL提供了大量通用算法，用于对容器中的元素进行操作，如查找、排序、复制、修改等。算法通过迭代器与容器交互，不依赖容器具体实现。包括：
非修改序列算法：如find、count、search等，不改变元素。
修改序列算法：如copy、replace、fill、remove等，会改变元素。
排序和相关操作：如sort、stable_sort、binary_search、merge等。
数值算法：如accumulate、inner_product等。

4、适配器：一种机制，用于改变容器、迭代器或函数对象的接口，使其适应不同的需求
（1）容器适配器
stack：后进先出（LIFO），默认基于deque实现，可指定底层容器stack<int,vector<int>>
queue：先进先出（FIFO），默认基于deque实现，需要支持push_back，pop_front
priority_queue：默认最大堆，基于vector实现，可自定比较器：priority_queue<int,vector<int>,greater<int>>
5、函数对象（仿函数）：行为类似函数的对象

重载了函数调用运算符()的类对象，可以像函数一样调用。可用于定制算法行为，如定义排序规则。STL提供了一些内置函数对象，如plus、minus、less、greater等。C++11后，lambda表达式也常用于替代函数对象。
6、分配器：内存管理组件
分配器用于管理容器的内存分配和释放。每个容器都有一个默认的分配器（std::allocator），使用new和delete进行内存管理。用户可自定义分配器以实现特定的内存管理策略，但通常使用默认即可。

/*
选择容器的决策流程：

1. 需要快速随机访问？
   - 是 → vector 或 deque
   - 否 → 进入2

2. 需要在中间频繁插入删除？
   - 是 → list 或 forward_list
   - 否 → 进入3

3. 需要元素自动排序？
   - 是 → set/map
   - 否 → 进入4

4. 需要快速查找（不关心顺序）？
   - 是 → unordered_set/unordered_map
   - 否 → 根据其他需求选择

特殊情况：
- 栈/队列 → stack/queue
- 优先队列 → priority_queue
- 固定大小 → array
*/
